## v1.0.0 · 初始版本

## v1.1.0 · 登录系统重构 & 教师班级管理

**发布日期**：2026-04-01

### 重大变更：学生登录方式

- **原方案**（存在 Bug）：用班级密钥 → 用 openid 查找学生 → 学生第一次登录时 openid 为空，永远找不到
- **新方案**：班级密钥 + **个人密钥（6位字母）** → 用 `classId + studentKey` 精确查找 → 首次登录自动绑定 openid

### 教师班级管理功能

1. 页面顶部展示班级密钥 + 教师密钥，支持一键复制
2. **补发密钥**：为旧学生（无 studentKey）自动生成个人密钥
   - 修复：前端直接 `db.update()` 受安全规则限制 → 新增 `fixStudentKeys` 云函数（管理员权限）
3. **导出密钥**：一键复制全部学生密钥到剪贴板
4. 学生卡片操作栏：移出班级 / 永久删除（二次确认 + 清除日志）

### Bug 修复

- 修复 tabBar：学生端不再显示教师标签占位
- 修复导入弹窗：点击 textarea 不再意外关闭

## v1.3.0 · 每日任务系统 & 数据库权限全面重构

**发布日期**：2026-04-01

### 核心问题发现 & 修复

**问题**：学生提交任务后教师端看不到，因为前端直接操作数据库受安全规则限制（只能读写自己创建的数据）。

**解决方案**：将所有前端直接数据库操作全部改为云函数调用（云函数有管理员权限）。

### 新增云函数

| 云函数 | 功能 |
|--------|------|
| `submitTask` | 学生提交任务完成 |
| `getPendingTasks` | 教师获取待确认任务列表 |
| `getStudentData` | 获取学生最新数据 |
| `login` | 统一处理教师/学生登录 |
| `getClassData` | 统一获取班级相关数据（学生列表、统计、日志、排行榜） |
| `updateStudent` | 更新学生信息（移出班级、赠予重置机会等） |
| `undoExp` | 撤回经验值操作（标记日志 undone + 扣除 EXP） |

### 前端改造范围

- `login.js`：改用 `login` 云函数
- `character.js`：改用 `submitTask` / `getStudentData` 云函数
- `teacher.js`：改用 `getClassData` 云函数
- `teacher-class.js`：改用 `getClassData` / `getPendingTasks` / `updateStudent` 云函数
- `teacher-score.js`：改用 `getClassData` 云函数
- `teacher-upload.js`：改用 `getClassData` 云函数
- `ranking.js`：改用 `getClassData` 云函数

### 每日任务功能

- 学生每天自动分配1个任务（可刷新3次）
- 任务完成需教师确认，奖励10 EXP
- 天赋匹配的任务额外 +5 EXP

### 近期操作 & 撤回

## v1.3.2 · 登录超时修复

**发布日期**：2026-04-02

### 问题

学生登录时控制台频繁出现 `Error: timeout`，但不影响使用。

### 原因

`_studentLogin` 方法连续调用了两次云函数：
1. 第一次：`login` 云函数获取 openid
2. 第二次：`login` 云函数进行实际登录

双重请求在网络稍有延迟时触发超时。

### 修复

- 前端移除单独的 `getOpenId` 请求
- 云函数 `studentLogin` 内部直接通过 `cloud.getWXContext().OPENID` 获取 openid
- 登录从 **2次** 云函数调用减少为 **1次**

### Bug 修复记录

#### Tab 切换白屏问题
- **问题**：点击 Tab 后页面卡住，所有操作无响应
- **原因**：
  1. `data-index="0"` 传的是字符串，但 `currentTab === 0` 比较时类型不匹配
  2. loading 遮罩层 z-index 过高覆盖了 Tab 导航
- **修复**：
  1. `switchTab` 中使用 `parseInt()` 确保索引是数字
  2. 移除 loading 遮罩层，改用下拉刷新

#### 任务确认显示失败问题
- **问题**：任务实际确认成功，但页面显示"失败"
- **原因**：前端判断 `res.result.success` 不够健壮
- **修复**：兼容多种成功返回格式

## 技术规范

- 云函数 `package.json` **必须**声明 `wx-server-sdk` 依赖
- 所有数据库写操作**必须**通过云函数（管理员权限），不能在前端直接操作
- expLogs 字段规范：`amount`（数字）/ `description`（字符串）/ `createdAt`（时间戳）/ `undone`（布尔）/ `type`（'score'/'class'/'task'）
- WXML `data-*` 属性永远是字符串，比较时需要 `parseInt()` 转换或使用 `==` 非严格相等
